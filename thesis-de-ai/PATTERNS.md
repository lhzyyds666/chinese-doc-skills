# AI 痕迹识别清单与改写示范

逐类列出 grep 关键词、识别要点、典型原句和改写后文本。原句均来自实战（红外卫星视频小目标跟踪本科论文），可直接套用到类似工程型论文。

---

## A. 模板化套话

**Grep**：

```
针对.{0,30}设计|针对.{0,30}问题.*?提出|本文(设计|构建|实现|达到|提出)|充分(说明|表明|体现)|为.{0,15}(提供|奠定).{0,10}(支撑|保障|基础|参考)|具有.{0,20}(性|意义|价值)|有效(性|地)
```

**典型原句 → 改写**：

| 原 | 改 |
|---|---|
| 第一，针对卫星红外原始数据是浮点辐亮度的特点，**设计了**基于高斯-拉普拉斯非锐化掩蔽的预处理流程 | 红外原始数据是浮点辐亮度，下游工具链都按 8-bit 三通道图像设计，因此预处理这一步用了高斯-拉普拉斯非锐化掩蔽 |
| 配套的图像解耦+重构机制实现了 MSE = 0 的可逆图像分解，**可作为后续可解释性分析与对抗样本生成的基础** | 重构 MSE 为 0，主要是为后续做可解释性分析时留出无损的中间产物 |
| **本文构建**了基于 DINOv2 主干 + 可变形 DETR 解码头的检测器 | 检测端选择了 DINOv2 主干配合可变形 DETR 解码头 |

要点：把"针对…设计了/构建了 X，达到 Y"的因果模板，拆成"现象/约束 → 因此 X → 实测 Y"的工程叙述。

## B. 工整三/四段并列

**Grep**：`第一[，,].{0,5}针对|第二[，,].{0,5}针对|第三[，,].{0,5}针对|第四[，,].{0,5}针对|第四[，,].{0,5}作为本文`

**典型问题**：8.1 工作总结/8.x 章节常见的"第一/二/三/四，针对…设计/构建/实现/提出了…"四段排比。

**改法**：完全打散并列结构，每段改为自然叙述（"现象 + 做法 + 结果"），首段不要写"论文的主要工作可以归纳为以下几点"，末段不要写"整体上…这一组合能够给出较稳定的结果"。

**改写示范**（4 段并列 → 4 段叙述）：

```
原（第一/二/三/四 模板）：
  第一，针对 X 的特点，设计了 Y 流程，从实测看…给下游模块提供了较高质量的输入。
  第二，针对 Z 的现实约束，设计了 …。
  第三，针对 W 的问题，构建了 …。
  第四，作为本文的主要工作，提出了 ……

改：
  红外原始数据是浮点辐亮度，下游工具链都按 8-bit 设计，因此预处理这一步用了 Y。负值数据和纯背景帧是开发过程中实际遇到过的两类异常输入，因此都加了对应分支……
  卫星视频数据本身就少，纯人工标注又慢，因此标注端走的是 ……
  检测端考虑的主要问题是小目标在深层特征图上经常变成亚像素，因此选择了 ……
  跟踪部分是本文的主要工作。整体沿用 ByteTrack 的双阈值二阶段关联，但在三个具体环节做了改动……
```

## C. 过满总结

**Grep**：`各项均|均达到或超过|证明了|显著提升|显著高于|效果显著|表现优异|起到.{0,10}(支撑|保障)|是.{0,10}主要原因`

**典型原句 → 改写**：

| 原 | 改 |
|---|---|
| 4 项指标都达到或超过阈值，**且差距比较明显**：…**说明算法**在小目标稀疏场景下的**稳定性较好** | 四项实测指标分别为 …，均达标。3 次独立测试得到的数字完全一致 |
| 这四点合在一起，对 90% 量级 MOTA 结果**起到一定支撑作用** | 从当前实验现象看，这几项设置与 AeroMOT 在本文测试集上的 MOTA 表现关系较为直接 |
| 三者**共同支撑了** MOTA 超过 90% 的**最终结果** | 这三处改动分别对应小目标召回、轨迹连续性和关联稳定性，最终 MOTA 达到 90.79% |
| 是 AeroMOT 在本文测试集上得到 90% 左右 MOTA 的**主要原因** | 与 AeroMOT 在本文测试集上的 MOTA 表现关系较为直接 |
| 两个核心相似度指标都**显著高于阈值** | 两个核心相似度指标均高于阈值 |

要点："证明""显著""主要原因"等强因果词全部降级；"各项均达标"改"几项指标均达到了预设阈值"；跨数据集（如本文 vs MOT17）禁止直接比较，要补"评测协议和数据集均不同，数字本身不可直接比较"。

## D. AI 归纳口吻

**Grep**：`综合看|可以看到|整体上|总的来说|综上所述|从.{0,10}可以看出|横向看下来|进一步|换言之`

**典型原句 → 改写**：

| 原 | 改 |
|---|---|
| **综合看**，把通用 MOT 算法迁移到小目标遥感场景… | （删去开头，直接进入论点） |
| **整体上**，"高分辨率单类检测器 + 贪心匹配 + 较保守的轨迹生命周期"这一组合**能够给出较稳定的跟踪结果** | 在本文测试集上，较高分辨率的单类检测器、按轨迹顺序的贪心匹配以及较长的 Lost 缓冲共同带来了较稳定的跟踪结果 |
| **横向看下来**，AeroMOT 大致是 ByteTrack 在小目标稀疏场景下的"减法版" | 从表 6-2 可以看出，AeroMOT 可以看作 ByteTrack 在小目标稀疏场景下的一种简化适配版本 |
| **可以看到** DETR 自动预标注 + OpenCV 人工修正这套半自动方案在像素级精度上较为稳定 | （删"可以看到"，直接陈述） |

要点："综合看""整体上""横向看下来"是 AI 总结句最强信号，删除或替换为"在本文测试集上""从表 X 可以看出"等具体引用。

## E. 投诚式解释 ⚠️ **必须删除**

**Grep**：`未做任何编造|数据真实可靠|绝无.{0,5}虚假|抠出|确实可信`

**为什么最危险**：这种主动声明真实性的句子，反而会让评审注意到数据真实性问题，是比 AI 痕迹本身更严重的风险。

**典型原句 → 改写**：

| 原 | 改 |
|---|---|
| 所有数值均从代码与 JSON 配置文件中**抠出，未做任何编造** | 相关数值均与代码实现和 JSON 配置文件保持一致 |

要点：永远不要在论文里主动解释"数据是真的"。

## F. 过度像教材

**识别**：在介绍方法时大段写"X 算法的原理是 …"而不写"本文为什么用 X，在系统中起什么作用"。

**改法**：把"教材式介绍"压缩，重点放在"本文采用该方法的理由 + 实现细节 + 在本系统中的作用"。

例：原文写满整段 "卡尔曼滤波的状态方程为 …，预测步 …，校正步 …"。改后保留必要的状态方程公式，但把方法解释压缩到 1–2 句，更多篇幅留给"本文用 8 维状态空间而非 SORT 的 7 维，是为了 …"。

## G. 隐性口语词

**Grep**：`搬到|跑下来|吐出|吐到|再压一压|试出来的|更值得相信|行得通|减法版|看起来是合理的|抠出|走完流水线|多绕一道|抢剩下的|交给.{0,5}给人`

**典型原句 → 改写**：

| 原 | 改 |
|---|---|
| 主流多目标跟踪算法**搬到这里**效果往往不理想 | 主流多目标跟踪算法直接迁移到该场景时，效果往往会受到限制 |
| 在本文红外测试集上**跑下来**，MOTA 为… | 在本文红外测试集上的实验结果为：MOTA 为… |
| **这一版**没有引入 Re-ID 分支 | 因此本文方法没有引入 Re-ID 分支 |
| 把数据**吐到**三个下游目录里 | 将数据整理到三个下游目录中 |
| 一次**吐出**三个下游消费端目录 | 最终生成三个下游模块所需的数据目录 |
| 也是后续可以**再压一压**的方向 | 也是后续仍有进一步优化空间的方向 |
| 这两个值都是在小目标场景下**试出来的** | 取值主要依据本文小目标数据上的调试结果 |
| 这和"高分检测**更值得相信**"的直觉一致 | 这与高置信度检测通常具有更高可靠性的经验判断一致 |
| 这一组合用于单类小目标检测**是行得通的** | 这一组合在本文数据集上的单类小目标检测任务中具有可用性 |
| 大致是 ByteTrack 在小目标稀疏场景下的"**减法版**" | 可以看作 ByteTrack 在小目标稀疏场景下的一种简化适配版本 |
| 这个取舍方向**看起来是合理的** | 这一取舍与本文卫星红外小目标场景的特点基本一致 |
| **走完流水线**之后派生出 16 个 crop 子场景 | 经过完整流水线处理后派生出 16 个 crop 子场景 |
| 把已经"看上去正确"的就近匹配**多绕一道** | 可能使原本较直接的近邻匹配被全局重排改变 |
| 老 ID 优先得到匹配，新轨迹**去抢剩下的检测** | 老 ID 优先得到匹配，新轨迹只在剩余检测框中进行关联 |
| 把绝大部分标框工作**交给模型完成**，只保留确实需要修正的部分**给人** | 由模型完成大部分初始框生成工作，再由人工对需要修正的标注进行精修 |

要点：这一类是"AI 改写后特有的痕迹"——AI 第一轮改完，往往会留下口语词以显得"自然"，反而更可疑。需要专门一轮处理。

## H. 重复词 / 改写残留

**识别**：同一句话或相邻句中关键短语重复出现。

**典型原句 → 改写**：

| 原 | 改 |
|---|---|
| **在本文数据集上**，DINOv2 主干 + 可变形 DETR 解码头 + Focal Loss 集合预测这一组合**在本文数据集上**的单类小目标检测任务中具有可用性 | 在本文数据集上，DINOv2 主干 + 可变形 DETR 解码头 + Focal Loss 集合预测这一组合对单类小目标检测任务具有可用性 |

要点：每轮改完通读一遍，前后句出现重复短语就删一处。AI 改写常因为前后句独立处理而引入这种重复。

---

## 收敛节奏（实战经验）

通常 3–5 轮可以收敛，每轮改 5–10 句：

1. **第 1 轮**：B + A + C（最严重的模板与过满总结）
2. **第 2 轮**：D + 第 1 轮残留的口语词（如"试出来的"）
3. **第 3 轮**：E（必删）+ 因果归纳过强的句子
4. **第 4 轮**：G 残留口语词 + 重复词 H
5. **第 5 轮**：定稿前再 grep 一次八类关键词全部清零

每轮先让用户读完反馈，再做下一轮，避免一次改太多导致"降 AI 模板"反而像新 AI 痕迹。
